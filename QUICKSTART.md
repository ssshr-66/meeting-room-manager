# 快速启动指南（Windows）

> 本指南适用于刚 clone 项目的 Windows 用户，帮助你快速启动前端进行演示验证。

---

## 📋 前置要求

- **Python 3.x**（推荐 3.8+）
- **现代浏览器**（Chrome / Edge / Firefox）
- **Git**（用于 clone 项目）

### 检查 Python 是否已安装

打开 **命令提示符（CMD）** 或 **PowerShell**，运行：

```bash
python --version
```

如果未安装，请访问 [Python 官网](https://www.python.org/downloads/) 下载安装。

> ⚠️ **注意**：安装时勾选 "Add Python to PATH"

---

## 🚀 启动步骤

### 1. 启动前端服务

在项目根目录下，运行以下命令：

```bash
cd frontend
python -m http.server 5500
```

看到以下输出表示启动成功：

```
Serving HTTP on :: port 5500 (http://[::]:5500/) ...
```

### 2. 打开浏览器访问

在浏览器中打开：

```
http://localhost:5500/
```

或

```
http://127.0.0.1:5500/
```

## 🔧 常见问题

### Q1: 启动后浏览器显示 "This site can't be reached"

**原因**：端口被占用或 Python HTTP 服务器未正常启动

**解决方案**：
1. 确认命令提示符中是否有 "Serving HTTP on" 的输出
2. 尝试换一个端口：
   ```bash
   python -m http.server 8080
   ```
   然后访问 `http://localhost:8080/`

### Q2: 点击登录后显示"网络异常"

**原因**：浏览器缓存或 Mock 脚本加载问题

**解决方案**：
1. 按 `Ctrl + Shift + R`（硬刷新）
2. 打开浏览器开发者工具（F12），查看 Console 是否有错误
3. 确认 `MR.Mock.enabled` 是否为 `true`（在 Console 中输入查看）

### Q3: 创建预约后刷新页面，数据消失

**原因**：这是正常的！Mock 数据存储在 localStorage

**解决方案**：
- 创建的数据会自动保存到 localStorage
- 如果数据确实丢失，可能是浏览器清除了缓存
- 可以在 Console 中运行 `MR.Mock.reset()` 重置到初始数据

### Q4: 想切换到真实后端 API

**解决方案**：
在浏览器 Console 中输入：
```javascript
MR.Mock.enabled = false
```

然后刷新页面。Mock 将被禁用，所有请求会发送到真实后端。

### Q5: 想重置所有演示数据

**解决方案**：
在浏览器 Console 中输入：
```javascript
MR.Mock.reset()
```

然后刷新页面，所有数据恢复到初始状态。
